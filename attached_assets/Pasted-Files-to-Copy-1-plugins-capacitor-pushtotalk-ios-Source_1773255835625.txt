1. plugins/capacitor-pushtotalk/ios/Sources/PushToTalkPlugin/PushToTalkPlugin.swift
Changes:

Trigger mode fixed: .click → .clickAndDoubleClick in both scan completion and managerDidRestoreState. This is the critical fix — .click mode suppresses double-click events, so your double-tap toggle was likely not working.
Queued events filtered: didReceiveButtonDown, didReceiveButtonUp, and didReceiveButtonDoubleClick now ignore queued events, preventing phantom PTT presses from replaying when the Flic reconnects.
Auto-reconnect: managerDidRestoreState now calls button.connect() on disconnected buttons so they reconnect automatically on app launch.
Auto-reconnect on disconnect: didDisconnectWithError now calls button.connect() (unless the button is unpaired) so it reconnects when it comes back in range.
forgetFlicButton method: New plugin method to unpair/remove a Flic button from the manager.
Bluetooth state tracking: Tracks FLICManagerState and checks it before scanning — returns a clear error message if BT is off/unauthorized.
Scanner status events: Emits flicScanStatus with granular progress (discovered, connected, verified, verificationFailed).
Connection failure events: didFailToConnectWithError now emits flicConnectionFailed to the frontend.
Unpair detection: Implements didUnpairWithError delegate, emits flicUnpaired.
Bluetooth state events: manager(_:didUpdateState:) now emits flicBluetoothState to the frontend.
